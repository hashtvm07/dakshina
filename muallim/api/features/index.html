Project Summary

Muallim Portal Completed Features

This document summarizes the features currently implemented in the Muallim portal codebase across the public website, admin panel, and NestJS API. It is based on the existing application routes, UI pages, services, entities, and seeded platform data.

3 Application layers
7 Admin modules
4 Public user flows
5 Core API domains

Public Website Features

The Angular public app exposes the main portal experience for visitors and Muallim applicants.

Landing Page

  • Dynamic hero section driven from admin-managed site settings.
  • About section with editable title, body text, hero image, and banner image.
  • Content blocks rendered in sort order from the backend.
  • Gallery/media section sourced from admin-managed media assets.
  • Latest vacancies preview shown directly on the homepage.

Muallim Registration

  • Public registration form with validation for name, username, email, phone, and address.
  • Optional bio field for profile details.
  • Successful registration returns and displays a generated Muallim ID.
  • Registration availability can be enabled or disabled from admin settings.

Vacancy Discovery

  • Dedicated vacancies page for browsing open positions.
  • Search and filter support by free text, subject, and location.
  • Search covers title, subject, location, qualification, contact info, and college name.
  • Each vacancy card shows college, subject, qualification, and contact information.

Muallim Public Profile

  • Profile route based on generated public ID.
  • Displays name, username, email, phone, address, and bio when available.
  • Public-friendly profile lookup handled directly through the API.

Admin Panel Features

The Angular admin app provides the operational interface for managing the platform.

Dashboard

  • Overview cards for total users, registered Muallims, colleges, and vacancies.
  • Single-screen operational summary for portal administrators.

Site Settings

  • Edit hero title, subtitle, description, about text, and image URLs.
  • Toggle the public registration menu and registration availability.
  • Changes are saved through the settings API and reflected on the public site.

Content Management

  • Create, edit, and delete landing page content blocks.
  • Manage media assets with title, category, and image URL.
  • Content blocks support manual sort order.
  • Public homepage content is fully data-driven from these records.

User Management

  • Create admin and operational users from the panel.
  • Assign user types such as superadmin, office admin, central admin, mekhala admin, and Muallim.
  • Assign visible menus/pages through selectable menu permissions.
  • Delete users directly from the user list.

College Management

  • Add, edit, and delete colleges.
  • Store college name, location, district, and state.
  • College records are reused while publishing vacancies.

Vacancy Management

  • Create, edit, and delete Muallim vacancies from the admin panel.
  • Capture title, subject, position count, location, qualification, last date, contact info, notes, and linked college.
  • Published vacancies automatically appear in the public website vacancy pages.

Registrations Monitoring

  • List registered Muallims inside admin.
  • Search registrations by Muallim ID or name.
  • Quick visibility into registered profile contact details.

Backend and Data Features

The NestJS API provides the data model and operational services behind both frontends.

Site API

Public payload endpoint, settings management, content block CRUD, and media asset CRUD.

Muallim API

Registration endpoint, public profile lookup by ID, and searchable Muallim listing for admin.

User API

User CRUD, metadata endpoint for user types and menu options, and automatic Muallim user creation on registration.

College API

College CRUD with reusable entity linkage for vacancy posting.

Vacancy API

Vacancy CRUD plus search filters for query, college, subject, and location with joined college data.

Health Endpoint

Base API status response with service name, status, and timestamp.

Bootstrap and Platform Setup

Implemented startup behavior that prepares the platform with default working data.

Default Superadmin

Bootstrap logic seeds a superadmin account and ensures a default password exists.

Default Site Settings

Hero copy, about content, and default images are seeded for first-run usability.

Default Landing Content

Initial content blocks and media assets are created automatically when the database is empty.

Runtime Config Support

Both Angular apps load API base URLs and endpoint mappings from runtime config JSON files.

PostgreSQL Integration

TypeORM is configured with PostgreSQL entities for users, Muallims, settings, content, media, colleges, and vacancies.

Role and Access Structure Implemented

User types and default menu visibility are already modeled in the backend.
